Section 04

Complete User Flow: Step-by-Step

FairLens is designed for a frictionless journey from raw data to actionable compliance report in under 3 minutes.

→

→
View Results

Download PDF
1
Register & Login (30 seconds)
User visits fairlens-unbiasedai.onrender.com → creates a free account with email/password. JWT token issued and stored in localStorage for persistent sessions.
Tech: FastAPI + bcrypt password hashing + python-jose JWT. Frontend: Next.js auth flow with form validation.
↓
2
Upload Dataset CSV
Drag-and-drop or browse to select a CSV or Excel file (up to 10 MB). The file is sent to the FastAPI backend via multipart form upload. Validation checks file format and size instantly.
Supported: .csv, .xlsx, .xls — pandas loads and validates the file. File is NOT stored permanently — processed in memory only.
↓
3
Intelligent Column Auto-Detection
FairLens automatically identifies sensitive attributes (gender, race, age, caste, religion, nationality, disability, marital status) and the target/decision column (approved, hired, outcome, label) using keyword matching on column names.
User can override detections. The UI shows all detected columns with checkboxes for confirmation before analysis runs.
↓
4
Bias Analysis Engine Runs (2–5 seconds)
The bias engine processes every selected sensitive column and computes 4 mathematical fairness metrics per attribute. Continuous numeric columns (age as float) are auto-binned into quartile groups. Minimum group size (5 rows) is validated to ensure statistical reliability.
Metrics computed: Demographic Parity Difference (DPD) | Equalized Odds Difference (EOD) | Disparate Impact Ratio (DIR) | Cramér's V bias contribution % | Most disadvantaged group identification
↓
5
Gemini AI Explanation (parallel, ~3 seconds)
Three Gemini API calls fire in parallel using asyncio.gather(): (1) explain_bias_findings — plain English analysis, (2) generate_fix_suggestions — prioritized remediation plan, (3) generate_report_summary — executive PDF summary. With 20-key API pooling, no single key exhausts during demos.
Key rotation: On 429/quota errors, system auto-rotates to next key. Supports up to 20 GEMINI_API_KEY_N environment variables.
↓
6
Audit Results Dashboard
Rich interactive results page shows: Overall fairness score (0–100), risk level badge (HIGH/MEDIUM/LOW), compliance panel (EU AI Act / EEOC / ISO 42001), bias drivers ranking, per-attribute metric bars, group comparison charts, most disadvantaged group callout, What-If Simulator, and full Gemini AI analysis.
Charts: Recharts library — horizontal bar chart for score overview, vertical bars for group positive rates. Framer Motion animations throughout.
↓
7
Download Professional PDF Report
One click generates and downloads a structured PDF audit report with: 2-column header with score and risk level, compliance table (EU AI Act / EEOC / ISO pass/fail), per-attribute metrics table, AI-generated summary, remediation recommendations, and footer with platform URL.
Tech: ReportLab library — professional table layouts, colored status indicators (green PASS / red FAIL), no emojis that break PDF rendering.

Regulatory Compliance Checks

⚖ EU AI Act
Article 10 & Annex III

High-risk AI systems must meet DPD < 0.1 and DIR > 0.8. FairLens auto-evaluates and reports PASS/FAIL with exact values. Enforcement begins 2026 — fines up to €30M.

📊 EEOC 4/5 Rule
US Equal Employment

Selection rate for any protected group must be ≥ 80% of the highest rate (DIR ≥ 0.80). FairLens checks and flags violations with exact DIR values.

📄 ISO 42001
AI Management System

Requires documented AI bias risk assessment. FairLens generates this documentation automatically — the PDF report serves as ISO 42001 evidence.

Ask Gemini: Interactive Q&A

After viewing results, users can ask Gemini free-form questions about their audit: "Who is being disadvantaged?", "What is the legal risk?", "Why is DPD high?" — Gemini answers in context of their specific dataset metrics. No hallucination risk — Gemini only sees the computed metrics, not the raw data.
